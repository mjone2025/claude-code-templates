# claude-code-templates 삽화 프롬프트

## Shot 1 - 템플릿 선택에서 진단까지

- 배치: 현행 워크플로우 문서 상단
- 인지 앵커: 필요한 컴포넌트를 고르고 CLI로 설치한 뒤 상태를 확인한다.
- 구조: 탐색 -> 컴포넌트 -> CLI 설치 -> 진단과 상태 확인
- 라벨: 탐색 / 컴포넌트 / CLI 설치 / 진단 / 상태 확인
- 민감정보: 프로젝트 식별자와 인증값 제외

## 안전 프롬프트

16:9 손그림 설명 삽화. 흰 배경, 검은 펜선, 넓은 여백. 왼쪽에는 탐색 보드, 가운데에는 agents, commands, hooks를 뜻하는 컴포넌트 상자, 오른쪽에는 CLI 설치 터미널을 둔다. 아래에는 진단 체크 막대를 둔다. 라벨은 `탐색`, `컴포넌트`, `CLI 설치`, `진단`, `상태 확인`만 쓴다. 프로젝트 식별자와 인증값은 넣지 않는다.
